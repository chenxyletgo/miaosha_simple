流程：
    有两个全局延迟队列
    程序启动就消费这两个延迟队列
    oderDelayedDelayQueue队列是所有生成的订单都会存入
        作用：验证订单是否超时了还没支付
    payMockDelayedDelayQueue队列是模拟用户支付用的
        给定随机时间，到时间了再随机判余额是否够等等
        作用：如果支付顺利就会修改订单支付状态，当支付
        状态为已支付后，当oderDelayedDelayQueue验证的
        时候就不会删除订单更新库存了，表示抢购成功了。

    测试记号.jmx是Jmeter测试配置文件，直接导入就ok了

    userIds.txt是Jmeter的接口数据来源的文件，可以直接导入后修改CSV的地址就可以了

    该程序只时候了解支付的超超超简步骤，像队列会不会撑爆
    等等都没有考虑，唯一考虑的就是超卖和限制支付时间这两个
    问题。新菜鸟在这感谢您的观看。
